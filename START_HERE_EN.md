## üí° How It Works

### Traffic Flow

```
1. Client connects ‚Üí wg0 (identify by source IP)
2. DNS query ‚Üí dnsmasq (resolve + add to ipset)
3. Traffic match ‚Üí iptables (mark traffic)
4. Route decision ‚Üí ip rule (select routing table)
5. Upstream forward ‚Üí wg-up-X (independent interface)
```

### Example: Domain-based Routing

```
Client A (10.8.0.2) visits google.com:

1. DNS query for google.com
   ‚Üì
2. dnsmasq intercepts and resolves ‚Üí 142.250.185.46
   ‚Üì
3. dnsmasq adds IP to ipset: client_1_proxy
   ‚Üì
4. Client accesses 142.250.185.46
   ‚Üì
5. iptables matches: -s 10.8.0.2 -m set --match-set client_1_proxy dst
   ‚Üì
6. Mark traffic: fwmark 101
   ‚Üì
7. ip rule matches: fwmark 101 ‚Üí table 101
   ‚Üì
8. Route via: default dev wg-up-1
   ‚Üì
9. Traffic goes through Upstream Server 1
```
